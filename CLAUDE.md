# CastCore — Project instructions

CastCore — **Stream. Manage. Control.** A self-hosted Streaming Operations Suite
(Python/FastAPI backend, React/Vite frontend, PostgreSQL, Redis, dedicated FFmpeg
Process Manager, Docker-first). Architecture: [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

## Binding rule: no feature change without a documentation update

Whenever you change **code, UI, API, data model, deployment, configuration, security,
CLI, installation or behaviour**, you MUST check whether documentation is affected and, if
so, update the matching **German *and* English** pages in the same change.

A change is not complete until its documentation is current. This is enforced by
[`CONTRIBUTING.md`](CONTRIBUTING.md), the documentation-rules page
([DE](docs/de/developer-guide/documentation-rules.md) ·
[EN](docs/en/developer-guide/documentation-rules.md)) and `scripts/check_docs.py` in CI.

### What to update, by change type

| You changed … | Update at least … |
| --- | --- |
| A user-facing feature/UI | `docs/{de,en}/user-guide/<area>.md` + screenshots |
| The API | `docs/{de,en}/api/<area>.md` + `developer-guide/api.md` |
| Data model / migration | `developer-guide/database.md`, `migrations.md`, manifest `models` |
| Deployment / Docker / proxy | `docs/{de,en}/admin-guide/...` |
| Config / env vars | `docs/{de,en}/reference/environment-variables.md`, `configuration.md` |
| Security / auth / secrets | `docs/{de,en}/admin-guide/security.md`, `secrets.md`, `reference/roles.md` |
| Error messages / codes | relevant `troubleshooting/*` pages + i18n |
| A new feature | new feature page from `docs/templates/`, add a `docs/docs-manifest.json` entry |

After doc edits, run the checker (regenerates `docs-status.json` + `nav.json`):

```bash
python scripts/check_docs.py        # 0 errors required; --strict also fails on warnings
```

If Python isn't available locally, run it in a container with the repo copied in (see how
it was bootstrapped) — CI runs it on every PR via `.github/workflows/docs-check.yml`.

## Docs system at a glance

- Source of truth: Markdown in `docs/de` (primary) and `docs/en` (parallel), structurally
  synchronized. Rendered in-app at `/docs` (no second build toolchain).
- `docs/docs-manifest.json` — feature → docs/UI/API/model mapping.
- `docs/docs-status.json`, `docs/nav.json` — generated by `scripts/check_docs.py`.
- Templates in `docs/templates/`.

## Conventions

- Keep DE and EN structurally identical (every page exists in both).
- No secrets in docs or examples; keep examples generic.
- In-app help: link UI areas to the matching docs page via the `HelpLink` component and
  the route→docs map in `frontend/src/lib/helpLinks.ts`.
